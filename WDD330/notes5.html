Ch 10: Testing and Debugging

  • Exception - gives value to help identify and solve the error. also produces stack trace

  • Errors:
    - System error
    - Programmer error
    - User error

  • strict mode - gives more exceptions/warnings, allows for more clear and organized code

  • the debugger keyword adds a breakpoint in code. remove after debugging

  • error objects:
    RangeError - number outside of allowed value range
    ReferenceError - reference made to non-existent item
    SyntaxError - error in syntax
    TypeError - wrong type of value used
    URIError - issue encoding/decoding URI
    InternalError - error in JavaScript engine

  • wrap code in try block to catch exception and pass into catch block.
    i.e.
    function example(){
    try{
    *function code here*
    }
    }
