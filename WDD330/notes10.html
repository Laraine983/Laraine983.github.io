Validating Forms

    What is form validation?

  • browser/server checks that data is in correct form requested by app/page
  • examples include error messages such as "please enter valid email address" or "this field is required" if data is not in correct format
  • client-side: validation done in browser
  • server-side: validation done in server

  • main purposes of validating forms:
    - ensure data is stored in proper format to maintain functionality of app
    - protect user data
    - protect ourselves

    Built-in validation

  • use within HTML form
  • required: Specifies whether a form field needs to be filled in before the form can be submitted
    minlength and maxlength: Specifies the minimum and maximum length of textual data (strings)
    min and max: Specifies the minimum and maximum values of numerical input types
    type: Specifies whether the data needs to be a number, an email address, or some other specific preset type
    pattern: Specifies a regular expression that defines a pattern the entered data needs to follow

  • CSS pseudo class :valid or :invalid - apply style to valid or invalid items

  • pattern attribute:
    a — Matches one character that is a (not b, not aa, and so on)
    abc — Matches a, followed by b, followed by c
    ab?c—Matches a, optionally followed by a single b, followed by c. ( ac or abc)
    ab*c—Matches a, optionally followed by any number of bs, followed by c. ( ac , abc, abbbbbc, and so on)
    a|b— Matches one character that is a or b
    abc|xyz — Matches exactly abc or exactly xyz (but not abcxyz or a or y, and so on)

    Constraint Validation API - JavaScript

  • HTMLButtonElement (represents a button element)
    HTMLFieldSetElement (represents a fieldset element)
    HTMLInputElement (represents an input element)
    HTMLOutputElement (represents an output element)
    HTMLSelectElement (represents a select element)
    HTMLTextAreaElement (represents a textarea element)

  • above elements can have following properties with the API:
    - validationMessage
    - validity
    - ValidityState
    - willValidate

  • methods:
    - checkValidity()
    - reportValidity()
    - preventDefault()
    - setCustomValidity(message)

    Using Fetch

  • Fetch API: provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses. It also provides a global fetch() method that provides an easy, logical way to fetch resources asynchronously across the network

  • Check for errors:
    fetch('flowers.jpg')
    .then(response => {
    if (!response.ok) {
    throw new Error('Network response was not OK');
    }
    return response.blob();
    })
    .then(myBlob => {
    myImage.src = URL.createObjectURL(myBlob);
    })
    .catch(error => {
    console.error('There has been a problem with your fetch operation:', error);
    });
