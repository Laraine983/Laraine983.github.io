Ch 11: Further Functions

    Function properties and methods

  • call() method - set the value of this> inside a function to an object that is provided as the first argument
  • if function does not use this in body, call() method must have null as first argument i.e. function.call(null, 5)
  • Custom properties
    - description (i.e. function.description) allows you to describe what function does
    - memoization - saves result in cache property to be used again later (i.e. function.cache)

    Immediately Invoked function expressions

  • function invokes as soon as defined. Use parenthesis at end of function i.e.
    (function(){ const cat = 'Tabby'; console.log(`Cute ${cat}`);})();
  • to use strict mode, place all code in IIFE (especially if collaborating with others) and use strict mode within that
  • allows you to write blocks of code that don't interfere with each other

    Recursive functions

  • invokes itself until condition is met

    Callbacks

  • functions passed to other functions as arguments and then invoked inside the function they are passed to
  • too many callbacks can create messy, confusing code

    Promises

  • represent the future result of an asynchronous operation
  • help simplify process when multiple callbacks used to avoid messy, confusing code
  • Life cycle:
    - pending - when promise calls asynchronous operation
    - unsettled - stage where operation is taking place
    - settled - operation has been completed. Outcomes of this can be:
    - Resolved - asynchronous operation successfully completed
    - Rejected - asynchronous operation not successfully completed

    Async functions

  • preceded by asynch keyword i.e. async function()
  • write async code as if it was synchronous

    Functions that return functions

  • just like using callbacks, functions can return other functions

    Closures

  • inner function returned by outer function allowing access to variables declared in enclosing function


    Ch 13: AJAX

  • AJAX = Asynchronous JavaScript and XML
  • Fetch API - used for sending/receiving data async across network. Uses promises to avoid callback issues
